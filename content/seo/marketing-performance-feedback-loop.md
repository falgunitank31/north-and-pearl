# Marketing Performance Feedback Loop

Date: 2026-07-21

Owner: Faraday

Purpose: turn organic marketing activity into measurable traffic, product-discovery, add-to-cart, and order improvements. This file defines when Faraday should ask other North & Pearl agents for changes based on verified performance data.

## Measurement Principle

Do not claim traffic, ranking, conversion, or order improvement until Bohr verifies the source data.

Required evidence sources:

- Shopify analytics
- GA4
- Google Search Console
- Bing Webmaster Tools
- Pinterest analytics
- Google Merchant Center free listings
- Shopify product and order reporting

## Weekly Review Rhythm

Every week, Faraday should review:

- Organic sessions by landing page
- Search impressions by collection and guide topic
- Organic clicks by query family
- Collection views from organic landing pages
- Product views from organic landing pages
- Add-to-cart rate by organic landing page
- Checkout starts from organic sessions
- Orders and revenue from organic sessions
- Pinterest outbound clicks
- Free Shopping impressions and clicks
- AI/referral traffic where analytics exposes it

## Decision Rules

### If a guide gets impressions but low clicks

Signal:

- Search Console impressions increase, but CTR stays weak.

Faraday action:

- Rewrite SEO title and meta description.
- Improve answer-style intro.
- Add clearer product/category angle.

Coordinate with:

- Kuhn if the page preview or visual presentation feels generic.
- Bohr to confirm the CTR baseline and date range.

### If a guide gets traffic but few product clicks

Signal:

- Organic sessions increase, but collection/product click-through is weak.

Faraday action:

- Add stronger internal links earlier in the guide.
- Add a tighter `Shop the guide` section.
- Clarify which collection fits which shopper intent.

Coordinate with:

- Kuhn for CTA layout and guide readability.
- Gauss if product examples are missing or weak.

### If collection pages get traffic but low add-to-cart

Signal:

- Collection organic sessions increase, but add-to-cart rate is weak.

Faraday action:

- Review search intent alignment and collection copy.
- Check if products shown match the collection promise.
- Recommend merchandising changes.

Coordinate with:

- Gauss for product sorting, product quality, images, naming, and catalog fit.
- Kuhn for product-card presentation.
- Bohr for add-to-cart and product-view diagnostics.

### If product pages get traffic but low add-to-cart

Signal:

- Product sessions/views increase, but add-to-cart is weak.

Faraday action:

- Review title, description, FAQ, gift positioning, and internal links.
- Check whether visitor intent matches the product.

Coordinate with:

- Gauss for product facts, supplier accuracy, image completeness, and product naming.
- Kuhn for product gallery, CTA visibility, and mobile UX.
- Wegener if shipping, returns, personalization, or support clarity is causing hesitation.

### If add-to-cart is healthy but orders are weak

Signal:

- Add-to-cart activity exists, but checkout starts or purchases lag.

Faraday action:

- Review trust content, product certainty, shipping visibility, return clarity, and gift confidence.

Coordinate with:

- Wegener for shipping/returns/support language.
- Kuhn for cart clarity and trust placement.
- Tesla for cart or checkout-adjacent technical issues.
- Bohr for funnel drop-off validation.

### If a product or collection gets no impressions

Signal:

- Search Console shows little or no visibility after a reasonable indexing window.

Faraday action:

- Improve internal links.
- Add or refresh collection copy.
- Add guide support content.
- Check canonical/indexing issues.

Coordinate with:

- Tesla for technical crawl/indexing checks.
- Bohr for Search Console validation.
