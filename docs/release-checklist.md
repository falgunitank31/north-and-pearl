# Release Checklist

Before pushing customer-facing changes:

- Inspect current implementation.
- Identify cross-functional impacts.
- Confirm no unsupported claims or internal notes are published.
- Run Theme Check for theme changes.
- Validate live storefront after deployment.
- Test product links, collection links, search, add to cart, cart drawer/cart page, and forms where affected.
- Check mobile rendering.
- Record files changed, Shopify settings changed, risks, assumptions, and missing owner inputs.
- Revert immediately if critical cart, product, or navigation behavior fails.
